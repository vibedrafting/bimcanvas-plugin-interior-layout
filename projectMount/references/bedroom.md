# 卧室策略

适用：tags 含 `sleep` 或 `bedroom` 的封闭空间。本策略文件指导**策略生成/施工**——从空间画像推导布局方案，并在布置阶段提供房间专项规范；不含流程步骤（流程由 generate 系列 Skill 管理）。

---

## 空间理解补充

在 generate-planning 产出的通用空间画像基础上，补充卧室特有分析：

从 `get_zone_boundaries` 返回的墙面清单构建候选（数据驱动，不需手动推断）：

1. **床头墙候选**：排除窗墙整面、排除门段区域（门洞+两侧各500mm），附候选墙实墙长度
2. **衣柜墙候选**：所有非窗墙面，有效长度 = 实墙总长 - exclusion zone / 床组占用 / 门侧净空扣减；不扣床头墙候选用的门段缓冲
3. **窗户朝向**：窗墙位置 → 决定"侧对窗户"的床头墙方向
4. **通道识别**（子分区场景）：passage 段 = 与相邻分区的开放连通，**禁止靠通道放大型家具**

> WHY：passage 在子分区场景中区分实墙和通道，靠通道放衣柜会阻断分区间通行。

## 策略生成

**执行模式判断**：
- **free mode**（无参考图）：按下述完整流程执行
- **constrained mode**（有参考图）：约束消费逻辑由 generate-planning 管理，本策略提供房间特有评估规则

**卧室空间节奏**（隐私梯度）：入口侧 = 更衣/收纳（过渡功能），远端/深处 = 床（私密功能）。
锚点决策（床头墙）和衣柜决策共同服务于这个隐私梯度——衣柜在入口侧创造过渡缓冲，床在深处保持私密性。不是两个独立的选墙问题，而是同一空间叙事的两个章节。

### 锚点决策：床头墙

**【必须】筛选**：排除窗墙 + 门段（门洞+两侧各500mm）→ 候选列表
**【必须】长度底线**：候选墙实墙长度 ≥ 2800mm（1800床+双床头柜的最低需求）；不足则降级检查 ≥ 2400mm（1500床+双床头柜）

**候选墙判定层级**：
- **合法候选**：未违反窗墙、门段、长度、禁区等硬约束。
- **推荐候选**：合法候选中更符合远离门、侧对窗、床脚不正对门等设计偏好者。
- **唯一合法候选**：硬约束筛选后只剩一面墙。

设计偏好只能排序，不能淘汰合法候选。若存在 ≥2 面合法候选，不得写"唯一合理睡眠墙"，只能写"推荐墙 + 备选墙"。

**若 ≥2 候选**：
- 当前执行者具备 `AskUserQuestion` → 逐面列出优劣势，向用户确认
- 当前执行者不具备 `AskUserQuestion` → 按当前推荐方案继续，并在结果中上报“卧室锚点自动代决”

设计偏好（构成推荐理由，**非淘汰依据**，按重要性排列）：
1. 远离门（隐私梯度：床在深处）
2. 侧对窗户（晨光温和 + 隐私）
3. 床脚不正对门（心理安全感）

> 注意：墙面长度是阈值型指标——超过底线后，4850mm 和 3300mm 对睡眠组没有差异，多出的长度不构成优势。不要因为一面墙"更长"就推荐它。当多面墙都满足长度底线时，设计偏好（远离门、侧对窗户等）才是区分优劣的依据。

> 注意：排除的是"门段"而非"整面门墙"——同一面墙上远离门的实墙段仍可作候选。

### 衣柜决策：选墙 → 选模式

**分区场景前置**：当本 zone 来自分区（边界含 passage 段），衣柜的墙面选择已在分区阶段做过全局审视。如果分区阶段将 wardrobeStorage 标签分配给了本 zone，遵循该决策；如果衣柜被分配到其他子 zone，本 zone 不再重复评估衣柜。

**模式判断**：**【建议】**优先评估 L 形，条件不满足时回退线性。

**L 形衣柜**（两种布局方式）：

| 布局方式 | 特征 | 空间效果 |
|---------|------|---------|
| **传统布局** | 沿墙段+垂直段（短边靠墙，长边伸入房间） | 垂直段形成隔断，创造半封闭换衣区 |
| **L型靠墙** | 所有段长边贴墙，沿墙角展开 | 最大化墙面利用率，适合紧凑空间 |

> WHY：传统布局通过垂直段创造"换衣区"——功能分区和使用体验更好。L型靠墙则最大化墙面利用率。不是好坏之分，而是空间组织策略的选择。

**布局方式选择**：默认传统布局。仅当垂直段阻挡通卫动线（末端到对面 < 900mm）时降级为 L型靠墙。垂直段方向远离卫生间门。

**线性布局**（回退方案）：
1. **【必须】**穷举 boundarySegments 中所有墙面，逐面评估
2. 排除窗墙整面 + 床组占据段
3. 计算有效段（按 exclusion zone、床组占用、门侧净空扣减；不得扣除床头墙候选专用的门段500mm缓冲）
4. 选有效段最长者
5. **【必须】**床前通道检查：若选定墙是床对面，房间进深-床深-柜深 < 800mm → 换墙
6. **【建议】**阵列重组填满：当标准柜体阵列后的剩余空间 < 定制柜体 min width 时，减少一个标准柜体、用多个定制柜体替代，使组合总宽填满有效段全长。例：有效段 3400mm，标准柜 1500mm → 2×1500+400(不足) → 重组为 1×1500+1×1200+1×700=3400mm

**【建议】**衣柜应填满有效段全长（标准柜体+定制补充）——连续柜面视觉整洁，最大化收纳。

**【建议】**有效段可容纳标准柜时优先标准柜；剩余 <600mm 且位于门口/通道侧时可作为操作净空保留，不应凭感觉退到更小定制柜。

WHY（→ 功能叙事目标）：连续衣柜面形成完整的"存储叙事墙"——换衣视线所及全是收纳，功能群组无断点；被梳妆台或斗柜插断的衣柜面讲述的是碎片故事，存储功能的叙事连续性被破坏。

### 家具配置

| 家具 | 主卧 | 次卧 | 策略参与 |
|------|------|------|----------|
| 床 | 必须 | 必须 | 锚点决策 |
| 衣柜 | 必须 | 有可用墙面时必须 | 主墙面决策 |
| 床头柜 | 优先成对（2个） | 优先成对（2个） | 跟随床 |
| 窗帘 | 有窗则必须 | 有窗则必须 | 跟随窗 |
| 梳妆台 | 可选 | 可选 | 位置探索 |
| 斗柜/展示柜 | 可选 | 可选 | 位置探索 |

> "位置探索"意味着策略阶段的方案探索中应包含该家具的候选位置分析，而非推迟到执行阶段。

### 睡眠组弹性组装

睡眠组沿床头墙排列：窗帘 → 窗侧床头柜 → 床 → 使用侧床头柜。

**床型默认策略（软默认）**：
- 主卧默认起点：`mod_bed_001 + 床头柜×2`（1800床 + 双柜）——主人长期使用，舒适度权重更高
- 次卧默认起点：`mod_bed_002 + 床头柜×2`（1500床 + 双柜）——客卧/儿童房/备用房复合使用，优先保留衣柜、窗帘、通道和可变功能弹性

WHY：床能不能放下不是唯一判断。主卧的核心价值是长期睡眠舒适，次卧的核心价值是均衡与弹性；若统一从大床起步，会让次卧过度占用空间，再把窗帘、衣柜和通道问题推迟到 fallback 才补救。

**【必须】窗侧锚优先**：双床头柜分支下，睡眠组整体必须贴窗侧锚布置。窗侧锚 = 窗帘盒/窗帘占位结束线之后，紧接窗侧床头柜；再接床；再接使用侧床头柜。剩余墙段只允许留在使用侧，不允许平均分配到窗侧和使用侧。

WHY：靠窗一侧不是视觉偏好，而是卧室空间节奏。睡眠区应占据采光深处，使用侧保留通行、衣柜和门段的弹性；若把睡眠组居中，窗帘盒与床之间会形成无功能空段，同时压缩使用侧的统筹空间。

卧室默认追求双床头柜完整睡眠组。空间不足时，**先换更小床，再减少床头柜数量**；禁止把 `1800 床 + 单床头柜` 当作常规降级方案。

**【必须】construction-brief 合同内弹性**：若主卧初始选择或次卧升级选择 `1800床 + 双床头柜`，`construction-brief` 必须在 `## 合同内 fallback` 章节写明睡眠组阶梯。若施工预检或 validate 发现它与满墙窗帘、必需衣柜、主要通道或门段净空冲突，允许 placement 自动降级为 `1500床 + 双床头柜` 并标记 `[自动适配]`；禁止自动改为 `1800床 + 单床头柜`；若需要进入 `1500床 + 单床头柜`、换墙、删除家具或降低衣柜等级，必须升级处理。

WHY：placement 只读 `construction-brief`。卧室策略里的确定性阶梯如果不被压缩进合同，施工阶段会把 `1500床 + 双床头柜` 误判为合同外改图。

**【必须】唯一合法睡眠墙锁定原则**：
- 只有硬约束筛选后只剩一面合法床头墙，才允许触发"唯一合法睡眠墙"锁定。
- 主卧：若唯一合法床头墙在满足边界、禁区、碰撞、满墙窗帘预留、通道与必需衣柜后，仍可合法容纳 `mod_bed_001 + 双床头柜`，则可锁定 `1800床 + 双床头柜`。
- 次卧：即使唯一合法床头墙可容纳 `mod_bed_001 + 双床头柜`，默认仍从 `mod_bed_002 + 双床头柜` 开始；只有空间非常宽裕且不破坏窗帘、衣柜、通道，或用户明确舒适优先时，才把 1800 写为升级方案。
- 若存在 ≥2 面合法床头墙，或 1800 双柜会迫使满墙窗帘截断、必需衣柜降级、主要通道不自然，则不得仅因床头墙长度足够而锁定 1800。

**次卧升级检查**：
次卧默认采用 `1500床 + 双床头柜`。仅当空间非常宽裕，且 `1800床 + 双床头柜` 不影响满墙窗帘、衣柜有效宽度、主要通道和可变功能弹性时，才允许把 1800 作为升级建议；若用户未明确舒适优先，不自动升级。

WHY：次卧的目标是完整睡眠组、合理收纳、窗帘完整、通道舒适和未来可变性；床头墙长度只证明床组本身放得下，不证明整个房间方案更好。

| 阶梯 | 组合 | 触发条件 | WHY |
|------|------|---------|-----|
| 1 | 1800床 + 双床头柜 | 主卧默认起点；次卧仅作为宽裕升级或用户明确偏好 | 主卧舒适度优先，次卧不自动占用弹性空间 |
| 2 | 1500床 + 双床头柜 | 次卧默认起点；主卧 1800 双柜不成立时 fallback | 保住完整睡眠组，同时释放收纳、窗帘和通道弹性 |
| 3 | 1500床 + 单床头柜 | 1500床 + 双柜仍放不下 | 作为紧凑卧室的最低合法组合 |
| 4 | 仅床 | 不作为常规方案 | 若连最低合法组合都不成立，应换墙或报告不可满足，而不是静默偷降 |

**【必须】定位算法**（按空间充裕程度分支）：

1. **计算可用长度**：床头墙有效实墙长度，扣除窗帘占位（窗侧）和相邻家具占位（使用侧）
2. **按房间身份选择默认起点**：主卧先尝试 `mod_bed_001 + 双柜`；次卧先尝试 `mod_bed_002 + 双柜`
3. **双柜分支组装**：默认组合若放得下且通过闭合预检，则从窗侧锚向使用侧连续组装：窗帘(200mm) → 窗侧柜 → 床 → 使用侧柜；组内 gap=0mm，剩余空间全部留在使用侧，禁止把睡眠组整体居中
4. **主卧 fallback / 次卧升级**：主卧 1800 双柜不成立时切换为 `mod_bed_002 + 双柜`；次卧 1500 双柜成立后，只有在空间非常宽裕且不破坏窗帘、衣柜、通道时，才可把 1800 写为升级建议或用户偏好方案
5. **若 1500 双柜仍不成立 → 改用 1500 单柜**：**从使用侧锚向窗侧顺序组装**——使用侧柜先定位（紧贴使用侧边界），`mod_bed_002` 紧贴使用侧柜，窗侧剩余间距归窗帘
6. **若 1500 单柜仍不成立**：不再降为 `1800床 + 单柜` 或 `仅床`；应回到床头墙选择或明确报告该房间无法满足最低睡眠组合

> **Stage 3 坐标计算**：按所选分支的组装方向，顺序累加各件尺寸即得坐标，组内 gap = 0mm（无缓冲）。construction-brief 必须明确写出窗侧锚坐标、组装方向、各构件连续区间、使用侧剩余段，以及 `mod_bed_001` 或 `mod_bed_002`，不能只写"床"。坐标由此算法产出，不从定性描述反推。

> WHY "从使用侧开始"：进入 `1500床 + 单柜` 分支后，使用侧柜成为唯一柜体——它的位置必须先确定（贴近使用侧边界，最大化使用侧通道），床再向它靠拢。如果仍从窗侧开始组装，使用侧柜的位置由"剩余空间"决定，可能被挤到不理想的位置。

> ⚠ 反模式：检测到空间紧张 → 直接把 `1800床 + 双柜` 降成单柜。正确做法：主卧先从 `1800床 + 双柜` fallback 到 `1500床 + 双柜`；次卧默认就从 `1500床 + 双柜` 开始；仍不足时才进入 `1500床 + 单柜`。床头柜与相邻衣柜的间隙是使用间隙（非通道），无最低通道宽度要求。

> ⚠ 反模式：存在多面合法床头墙时，把"推荐墙"写成"唯一合理睡眠墙"，再用 1800 双柜锁死整个布局。正确做法：先保留合法候选，再用联动检查判断床型、衣柜、窗帘和通道的综合质量。

### 可选家具

梳妆台、斗柜、展示柜、落地灯等。

**策略阶段**：**【必须】**为每件可选家具独立扫描所有墙段（含异形区），标注候选墙面及理由——在评估主要家具布局之前或同时完成，不在主要家具锁定之后。

> ⚠ 反模式："先确定衣柜占据哪些墙段 → 在剩余墙段中为梳妆台找位置"。这会让大件的布局决策预先消除小件的候选位置，制造虚假的"只有一个可行位"。**诊断信号**：如果可选家具最终是孤立放在一面空墙上（不与其他家具组合），检查该位置是否因为"其他墙段被占满"而非"该位置最优"。正确做法：全部家具的候选墙面同时列出，再做全局权衡。

**L 形卧室的异形区候选定义**：异形区不是只看窗边或凹角留下的短墙。L 形、T 形、凹位空间中，以下墙段都应进入可选家具候选池：
- 凹角两侧短边、延伸段端墙、转角相邻墙。
- 与入口、更衣/收纳、卫浴动线形成连续功能带的墙段。
- 与主家具形成 L 形/并排工作站的墙段，即使它不在采光最好的主体区。

WHY：异形区的价值常来自功能组织，而不只是残余尺寸。若只扫描窗边短墙，Agent 会漏掉"收纳 + 可选功能"形成工作站的转角方案。

**联动检查**：当可选家具的候选位置均违反其 relation_rules 中的多条【建议】时（如：远离窗户、孤立布置、侵占通道），检查是否因主要家具（衣柜）占据了更优位置。若是，评估"调整主要家具为可选家具腾位"的代价与收益——缩短衣柜一段（牺牲收纳）换取梳妆台获得组合位置或采光位置，通常是值得的。

WHY：可选家具的位置可能改变主要家具的最优布局。全局同步探索才能发现跨优先级的联动优化。

**【必须】墙段搜索顺序**：先异形区墙段 → 后主体区墙段。两个区域的候选都应列出，而非找到即停。

WHY："优先"控制的是搜索顺序（先看哪里），不是选择排他（找到就不看别处）。异形区可能有空间组合优势，主体区可能有采光/动线优势——两者各有所长。当异形区和主体区都有可行候选，且它们在【建议】规则上存在显著差异（如采光 vs 组合布置），这是战略选择信号，应向用户展示。

只要通道允许（次通道≥600mm），可用墙段都应被利用。

**候选类型标注**：梳妆台等主动使用类家具至少按以下类型标注候选。**每种候选必须同时评估三个维度**——采光、组合便利、**动线侵占成本**：

- **采光型候选**：靠近窗、自然光更好。
  - 动线侵占成本：若位于"走线末端"（如床尾对面墙的窗侧端、房间深尽端），前方使用空间对主通道侵占为零；若位于墙段中段，需评估前方使用空间是否落在主动线路径上。
  - 典型缺点：可能与收纳或更衣动线割裂。

- **组合型候选**：与衣柜、收纳柜、卫浴动线或入口前场形成连续工作站。
  - 动线侵占成本：紧邻衣柜的组合位常落在"床区→主卫"或"入口→床区"动线上，需检查前方 600mm 使用空间 + 家具深度是否压缩通道宽度。**视觉连续 ≠ 动线无侵占**——紧邻衣柜的"组合"如果占据动线关键位置，反而不如走线末端的"独立"位。
  - 典型缺点：采光可能较弱。

若采光型与组合型都几何可行，**按"采光 + 组合便利 + 动线侵占"三维综合权衡**：若组合型候选侵占主通道（前方 1000mm 使用区落在动线上）、且采光型候选位于走线末端，**采光型优先**——这也是 `module_library.json` 中"避免孤立布置"规则的"床尾对面墙靠窗端例外"的内部机制（采光好 + 走线末端 = 动线侵占为零）。

若三维综合后仍优劣相当，这是战略选择信号。

#### 反例：组合型紧邻衣柜但侵占主通道

床头墙=西墙、床尾朝东、主卫门在 NE 角延伸区。L 形衣柜在 NE 角包裹（北墙东段 + 东墙₂上端）。梳妆台候选评估：

❌ **错误选位**：东墙₁ 中段（紧邻 L 形衣柜垂直段下方）
- 表面：紧邻衣柜，"视觉连续"，看起来是组合型优势
- 实际：前方 600mm 使用空间 + 400mm 梳妆台深度 = 1000mm 占用，**正好落在"床区→主卫"动线上**
- 后果：床区到主卫的人需绕开梳妆椅，通道宽度被硬性压缩

#### 正例：采光型在走线末端、动线零侵占

✅ **正确选位**：东墙₁ 窗侧端（靠南窗）
- 靠南窗 → 采光最优
- 位于房间最南端尽端 → 走线末端
- 床区→主卫的人不会经过此处 → 动线侵占为零
- 即使看似"独自占一面墙的一小部分"，也不构成"孤立布置"——"避免孤立布置"反模式的核心条件是"独占 + 占小部分 + **侵占主通道**"三项同时成立，本位置不满足第三条

WHY："孤立布置"反模式的核心负面表现是"侵占主通道"，不是"独占墙段一小部分"。走线末端的位置即使独占小段墙面，也不产生孤立布置的实际危害——反而获得最佳采光 + 零动线侵占。

**功能组合优先**：可选家具先评估与主家具、固定功能带、入口/卫浴动线的组合潜力。梳妆台 + 衣柜属于一级组合候选：它们能形成完整梳妆收纳工作站（伸手即取），不要求两者都属于可选家具。同功能区内的家具应优先形成空间组合（L 形/并排），再为组合整体选位，而非逐件独立匹配墙段。

WHY：化妆台+收纳柜形成 L 形组合 = 完整梳妆工作站（伸手即取）；散落两面墙 = 两件孤立家具。组合带来的功能协同 > 单件家具的"最优墙段"。

**卧室领域战略选择信号**：
- 床头墙多合法候选，按锚点决策规则处理。
- 可选家具出现采光型候选 vs 组合型候选，且都几何可行。
- 同一功能带存在"纯收纳"与"收纳 + 可选功能工作站"等不同日常体验。
- 可选家具位置会改变衣柜、卫浴动线、睡眠区留白的叙事关系。

以上只标记为战略选择；是否 Ask 由当前执行者的交互能力决定。

**L 形组合内部位置分配**：当两件可选家具形成 L 形组合时，角落位置分配给较高/较深的家具（如收纳柜），开敞侧分配给需要主动使用的家具（如梳妆台）。

WHY：角落光线暗、可达性差——高柜体（被动存取，开门即用）不受影响；梳妆台（主动使用，需要坐下操作）需要光线和空间感。

**尺寸最大化**：可选家具是 parametric 模块，宽度在 limits 范围内可调。确定位置后，按以下步骤决定宽度：
1. 计算该墙段有效长度（扣除 exclusion、门侧净空、相邻家具占用）
2. 取 min(有效长度, limits.width.max) 作为目标宽度
3. 若模块定义了 morphology.step，向下取整到 step 整数倍（如 step=100 时 target=950 → 取 900）
4. 执行顶角规则检查（两端间隙 < 600mm 且非门口/通道侧 → 扩展或平移消除）

WHY：可选家具默认 size 是最小值，Agent 容易"凭感觉"选中间值而非最优值。显式步骤确保尺寸由空间决定而非直觉。

## 策略推导示例

> 以下示例展示从空间画像到布局方案的完整推导。参照此密度和推理风格，用当前空间数据独立推导。

### L 形卧室分区示例：衣帽区 + 睡眠区

**场景**：L 形主卧，主体约 3300×4850mm，东北角延伸约 1700×1550mm；tags=[sleep, wardrobeStorage]，optionalTags=[vanity]；北墙入口，延伸区靠近卫浴动线。

**分区预演**：
- 主要家具粗占位：睡眠组需要方正安静区；衣柜深 600mm，前方需要约 900mm 操作/通行净空。
- 通道/留白识别：入口 → 衣柜/卫浴 → 睡眠区应形成隐私递进；衣柜前方净空是更衣操作区，不是可继续塞家具的剩余面积。
- 边界浮现：北侧/东北延伸优先形成更衣区，南侧主体形成睡眠区；边界可贴合 L 形转折，不必用两个矩形铺满整个卧室。

**推荐分区**：
- `dz_1` 更衣区：tags=[wardrobeStorage]，optionalTags=[vanity]。沿北墙或延伸区墙面组织衣柜，必要时保留衣柜前操作净空；若短边采光/动线合适，可容纳梳妆台。
- `dz_2` 睡眠区：tags=[sleep]。放在更深处的方正主体区，保证床两侧使用和床尾通行。
- 有意留白：入口缓冲、衣柜前操作净空、通往卫浴/睡眠区的主通道不生成独立 designable 子区，后续禁止被大型家具占用。

**可选功能候选对比**：
- 候选 A（采光型）：主体区西墙下段靠窗，采光好，适合独立梳妆；缺点是与入口侧衣柜割裂，且会占用床脚侧留白。
- 候选 B（组合型）：东北延伸段端墙或转角相邻墙，与北墙衣柜形成 L 形梳妆收纳工作站，并靠近卫浴动线；缺点是自然采光弱，需要校验卫浴通道和操作净空。
- 结论：A 与 B 都几何可行且影响日常体验，这是战略选择。主控 Agent 应 AskUserQuestion；layout-agent 按推荐方案自动代决并上报“可选功能自动代决”。

**取舍判断**：
- 若把 L 形强拆成左右两个矩形，容易让主体区同时承担床和衣柜，压缩主通道；延伸区又变成无明确功能的小块。
- 若把北侧/延伸区组织成更衣区，衣柜、梳妆和卫浴动线形成连续叙事，睡眠区保持安静方正。

WHY：卧室分区服务于隐私梯度和更衣叙事，不是几何切割。衣柜前净空和入口缓冲是卧室体验的一部分，不能因为没有 tags 就被视为需要填满的空地。

**场景**：3.6m × 4.2m 主卧，南墙窗户，北墙入口偏左

**空间画像摘要**：
```
动线：北墙偏左入口 → 南向到窗
纵深：北(入口/前场) → 南(窗/深处)
采光：南墙窗户，采光轴南北

get_zone_boundaries 返回：
  北墙 | 总长 4000mm | 实墙 3200mm
    wall 1200mm ... + door(d_1) 800mm ... + wall 2000mm ...
  东墙 | 总长 3600mm | 完整实墙
  南墙 | 总长 3600mm | 实墙 800mm（窗墙）
    wall 400mm ... + window(wi_1) 2800mm ... + wall 400mm ...
  西墙 | 总长 3600mm | 完整实墙

床头墙候选（排除窗墙 + 门段缓冲）：
  东墙 3.6m 完整实墙（最长）| 西墙 3.6m 完整实墙 | 北墙右侧约 2.0m（扣门段500mm缓冲）

衣柜墙候选：
  北墙右侧 2.0m | 东墙 3.6m（若不给床）| 西墙 3.6m（若不给床）
```

**策略推导**：
- **床头墙** → 东墙：长度充足(3600mm≥2800mm) ✓ 侧对南窗 ✓ 远离北墙门 ✓ 不正对门 ✓
- **衣柜** → 北墙右侧 2.4m + 西墙北端形成 L 形：传统布局，垂直段从西墙北端向南伸出（远离北墙门，不阻通卫），创造入口侧半封闭换衣区
- **间距** → 窗侧=窗帘200mm+床头柜深度，使用侧=床头柜深度+剩余全部
- **窗侧锚坐标** → 若南窗窗帘占位为 `Y=900→1100`，则睡眠组从 `Y=1100` 后连续展开：窗侧床头柜 → 床 → 使用侧床头柜，剩余空间全部留在北侧/使用侧
  - 正确：窗帘盒结束线紧接窗侧床头柜，睡眠组贴窗侧锚
  - 错误：睡眠组整体居中，窗帘盒到窗侧床头柜之间留出大空段
- **可选家具** → 梳妆台：
  - 候选①采光型：西墙南段，靠窗采光 ✓，独立梳妆体验 ✓，但与入口侧衣柜组合弱 ✗
  - 候选②组合型：北侧/延伸段转角，和衣柜形成 L 形梳妆收纳工作站 ✓，靠近更衣/卫浴动线 ✓，但自然采光弱 ✗
  - 两个候选均几何可行且影响日常体验：主控 Agent → AskUserQuestion；layout-agent → 按推荐方案继续，并上报“可选功能自动代决”
- **设计目标**：入口侧衣柜形成连续存储叙事墙(功能叙事) / 入口→换衣区→床的隐私递进(空间节奏) / 梳妆台在采光型与组合型候选之间按用户偏好或自动代决收束
- **配置** → 主卧默认 `mod_bed_001` 1800床 + L形衣柜 + 床头柜×2 + 窗帘 + 梳妆台（位置来自已确认候选）
- **说明**：该例为主卧，舒适度默认权重高；闭合预检确认 `1800床 + 双床头柜` 不破坏窗帘、衣柜、通道等关键目标，因此无需降级为 `1500床 + 双床头柜`

**紧凑次卧对比示例**：2.8m × 3.4m 次卧，北墙窗、南墙偏东门，西墙与东墙均为合法床头墙。

- **床头墙**：西墙更符合远离门，可作为推荐墙；东墙仍是合法备选，不能写成"西墙唯一合理"。
- **衣柜有效段**：南墙门西侧 1870mm，次卧门侧净空默认 300mm → 有效段约 1570mm，可优先考虑 1500 标准柜；不得再扣床头墙门段 500mm 缓冲。
- **睡眠组默认**：次卧从 `mod_bed_002 + 双床头柜` 开始。若 1500 双柜能保留满墙窗帘、1500 衣柜和自然通道，应推荐 1500 双柜并上报"次卧平衡代决"。
- **1800升级条件**：只有当 `1800床 + 双床头柜` 不截断满墙窗帘、不迫使衣柜退到 1200mm、不压缩自然通道，且用户明确舒适优先或空间极宽裕时，才写为升级建议。

## 关键约束速查

**硬约束**：
- 床头禁靠窗墙 — 光线噪音+心理不安全
- 穷举所有墙段评估衣柜 — 禁止跳过任何候选（含异形区墙段）
- L型靠墙内角门开启检查 — 按方案A(移门)→B(足够外露)顺序评估

**软指导**：
- 衣柜填满有效段全长 — 连续柜面+最大化收纳
- 侧对窗户 — 晨光温和+隐私
- 成对床头柜（主卧）— 视觉对称+双人便利
- 大型家具贴靠实墙角 — 窄缝积灰难清洁
