# 布置设计原则

> 跨房间通用知识。房间特定策略在各房间 Skill 中定义。
> 约束层级：**【必须】**不可违反 · **【建议】**默认遵守，可说明理由后偏离 · **【提示】**鼓励灵活处理

## 一、设计原则速查

| 规则 | WHY |
|------|-----|
| **【建议】靠墙** | 释放中央活动区域，减少碎片化 |
| **【建议】居中** | 视觉均衡（电视柜居中电视墙、床居中床头墙） |
| **【建议】顶角** | 大家具贴靠实墙角，窄缝(<600mm)积灰成卫生死角 |
| **【建议】朝向** | 背面靠墙=心理安全感，正面朝空间=便于使用 |
| **【建议】对位** | 相关家具功能对应（沙发对电视、餐椅围餐桌） |
| **【建议】避窗** | 窗前大家具遮光且压缩空间感（卫生间淋浴区例外） |
| **【必须】避门** | 不阻挡门开启范围——门扇受阻导致功能丧失 |
| **【建议】前瞻协调** | 放主家具时预判可选家具的策略位置，避免阻断其空间需求 |
| **【必须】优先级省略** | 空间冲突迫使省略组内元素时，按优先级从低到高省略——不按物理位置截断末端。截断末端 = 让组装方向决定设计取舍，而非设计意图决定 |

> 各规则适用家具由 module_library agent_config 标注，此处不重复。

## 二、全局规划

**空间阅读三要素**：
1. **动线方向**：入口到各功能区的行走路线。主通道(≥900mm)连贯无阻断——动线断裂导致日常走动反复绕行
2. **纵深层次**：入口侧="前场"（过渡/储物），远端="深处"（私密/休息），密度前高后低——渐进节奏创造空间体验
3. **采光轴**：窗户是"呼吸面"，避免高大家具切断窗→深处的光线通路——遮挡让空间变暗变压抑

**功能分区**：封闭空间通常 2-3 个功能区，各区有"锚定墙面"，家具群组围绕此墙面组织。用动线自然分隔，而非家具硬隔断。

## 三、布置优先级

锚点 → 主要 → 可选。先确定功能核心，再围绕它组织空间。

| 层级 | 定义 | 位置特征 |
|------|------|---------|
| 锚点 | 功能核心 | 由空间决定，几乎无选择 |
| 主要 | 功能必须 | 由锚点+空间共同决定 |
| 可选 | 功能补充 | 策略阶段同步探索位置 |

WHY：优先级决定执行顺序，但所有家具在策略阶段同步探索——可选家具的位置可能影响主要家具的最优布局。

> 各房间的锚点/主要/可选清单和必须/可选家具在各房间 Skill 中定义。

## 四、空间硬约束

**【必须】**任何家具都不可违反——违反任何一条都会导致 validate_layout 报错：
1. bounds 完全在 zone.innerBoundary 内——超出边界的家具在实际空间中会穿墙
2. bounds 不与 zone.exclusionAreas 重叠——禁区是柱子、管道等不可占用区域
3. bounds 不与其他已放置模块重叠——物理空间中两个物体不能占据同一位置
4. 不阻挡门的正常开启——门扇受阻导致功能丧失

> innerBoundary 已扣除墙面完成面厚度，直接以它为可用范围，**不要**再额外预留。

## 五、通道与间距

| 通道类型 | 最小宽度 | 典型场景 |
|----------|---------|---------|
| 主通道 | ≥ 900mm | 入口→核心区主动线 |
| 次通道 | ≥ 600mm | 家具间穿行、床脚到墙 |
| 床侧（使用侧） | ≥ 500mm | 日常上下床 |
| 床侧（窗侧） | ≥ 200mm | 仅预留窗帘空间 |

**通行间隙 vs 使用间隙**：通道标准只适用于**人需要从此处走过到达其他区域**的间隙。人在旁边伸手操作即可的间隙（如床头柜与相邻衣柜之间）无最低通道要求。WHY：区分通行和使用，避免所有间隙都按最大标准留而浪费空间。

**通用使用距离**：餐椅后退 600-800mm（起身活动）· 书桌椅后退 800-1000mm（起身+通行）

**门侧净空**：**【建议】**门所在墙布置柜体时，门洞旁至少保留一侧可操作净墙。次卧默认 ≥300mm，主卧默认 ≥450mm；紧凑空间可压缩一档，次卧不低于 200mm，主卧不低于 300mm。该净空仅用于开关/操作，不等同于床头墙筛选中的“门段 500mm 缓冲”。

> 房间特定使用距离（电视观看距离、衣柜开门空间等）在各房间 Skill 中定义。

## 六、家具依赖

**成套家具**——同组必须同阶段放置，保证空间关系协调：

| 依赖组 | 组成 | WHY |
|--------|------|-----|
| 睡眠组 | 床 + 床头柜(1~2) | 床头柜紧贴床头两侧 |
| 客厅组 | 沙发 + 茶几 | 功能对位关系 |
| 餐厅组 | 餐桌 + 餐椅(N) | 围合就座关系 |
| 工作组 | 书桌 + 椅子 | 操作位关系 |

**功能互斥**：同一区域不重复放置同功能家具（如两张床），儿童房双床除外。

> 各房间的必须/可选清单在各房间 Skill 中定义。

## 七、常见错误速查

| 错误 | 正确做法 | WHY |
|------|---------|-----|
| 家具挡门 | 检查门扇弧线区域 | 门无法打开=功能丧失 |
| 通道过窄 | 主通道≥900mm，次通道≥600mm | 人体通行最低需求 |
| 沿墙留卫生死角 | 贴靠实墙角，空隙留在通道侧 | <600mm窄缝积灰无法利用 |
| 重复扣减完成面 | innerBoundary 已扣除，直接用 | 二次扣减导致可用空间白白缩小 |
| 成套家具坐标逐件估算 | 组装锚+顺序累加：锚坐标 → 依次加各件尺寸，每件 gap = 0mm | 逐件估算在紧贴处引入安全缓冲（AI 直觉），积累成肉眼可见缝隙；顺序累加是闭合系统，结构性消除缝隙 |

## 八、修正优先级

验证失败时按以下优先级修正——从最小变动到最大变动：

**平移** → **旋转** → **缩小**（limits 内 + step 对齐）→ **拆除附属件** → **替换**同功能更小模块 → **移除**

**拆除附属件**：当碰撞的是附属件（如梳妆台的椅子、书桌的椅子）而非主体时，优先移除附属件、保留主体位置不变。现实中椅子可收纳到桌下，移除附属件的代价远小于迁移主体。

WHY：平移保留原始设计意图，移除是最后手段。每步都在保护前序决策的成果——只有当更小变动无法解决冲突时，才升级到更大变动。附属件移除是比替换/移除主体更小的变动——用户失去一把椅子比失去整个梳妆台位置更可接受。
