# 分区 Skill

> 为空间制定布局策略：决定这个空间应该有几种不同的体验,它们之间是什么关系。
> 分区是"规划"的组成部分——空间功能定义后，后续房间策略在此框架内精确布置。

**与 generate-planning 的协调**：本 Skill 在 Stage 2.2（分区设计）被加载。此时 generate-planning 已通过 `save_semantic_plan(spatial-skeleton)` 提交了：
- 语义方案 spatial-skeleton（空间骨架：动线/纵深/采光/初步意图）
- 基于物理约束确定的主要家具最优墙面位置

**输入**：spatial-skeleton 已提交的空间骨架 + 主要家具墙面结论
**输出**：分区兼容性结论 + 功能定义（写入 strategic-plan）
**不做**：不重新分析空间几何、不重新推导墙面语义、不重做主家具决策

本 Skill 基于这些已有分析做分区评估——不重新推导空间信息。执行顺序：先用已确定的家具位置过滤不兼容的分区方案，再做功能定义。

**禁止重新评估家具位置**：家具的墙面归属已由房间策略确定，分区评估只检查"分区方案是否兼容这些已确定的需求"。

WHY：分区赋予空间以意图。有意图的空间才能产出有设计感的布置——家具属于各自区域的功能叙事，区域间的留白是有意识的空间节奏。没有分区的布置只是"把家具塞进房间"。同时，分区解耦约束，让每个子空间聚焦单一功能，降低布置难度。

## 一、本 Skill 的职责

本 Skill 始终被加载，负责两件事：
1. **空间预演**（步骤 1）：评估空间是否需要物理分割，如需则确定分割方式
2. **功能定义**（步骤 2）：为空间（或各子空间）定义功能身份

"不分区"是合法结论——步骤 1 判定无需分割后，步骤 2 仍需执行，输出空间内部的语义功能带；若空间确实只有一种体验，再定义为单一功能区。

WHY：功能定义是布置的前提。每块空间的功能身份在策略阶段之前就应该明确——否则后续家具布置是在没有空间意图的基础上进行的。分区 Skill 的核心产出不是 subZones 数据，而是"空间各部分承担什么功能"的显式决策。

## 二、分区方法

### 步骤 1：空间预演（粗占位 + 通道/留白识别）

分区线不是凭空画出来的，也不是先把房间切成矩形后再填家具。它从主要家具粗占位、人的通行路径和有意留白中浮现。

**快速判断**：规则空间 + 单功能标签组 + 主要家具粗占位无冲突 + 主通道自然成立 → 结论"无需物理分割"，直接进入步骤 2。

**需要详细评估的信号**（命中任一则进入粗占位预演）：

| 信号 | 判断依据 |
|------|---------|
| 异形空间 | rawBoundary 顶点 > 4，存在凹角、转折、延伸段或可独立围合的空间体验 |
| 多功能组合 | 多个功能标签需要不同锚点、不同采光/动线条件或不同私密层级 |
| 功能间冲突 | 已确定的主要家具墙面在同一空间内压缩主通道、操作净空或采光轴 |
| 空间比例失调 | 单一功能区会造成动线过长、功能分散或大量无意残留 |

**1a. 主要家具粗占位**：基于已确定的主要家具墙面结论，估算主要家具占用带和必要操作净空。只验证"能不能形成空间组织"，不重新选择家具墙面。

**1b. 通道/留白识别**：在粗占位之后，先识别入口到各功能点的主通道、入口缓冲、采光留白、操作净空和无效窄缝。

- 通道、入口缓冲、采光留白、操作净空是有意留白，不是遗漏空间。
- 有意留白不强制生成 `designable` 子区；它应在分区分析和语义方案中说明位置与目的。
- 若某块留白需要承载家具或明确功能，才把它定义为功能子区并映射 tags。

WHY：当前错误来自把"铺满父空间"当成分区目标。真实分区组织的是使用体验：家具占哪里、人从哪里走、哪里必须空出来。没有目的的残留是问题；有目的的留白是设计。

**1c. 功能边界浮现**：从主要家具占用、通道/留白和建筑轮廓中推导功能子区边界。

- 边界优先贴合建筑轮廓和功能占用，而不是优先简化为矩形。
- 在不增加无意义复杂度的前提下，可使用正交多边形贴合 L/T/凹角空间。
- 矩形只在空间本身接近矩形，或功能占用确实形成矩形区域时使用。
- 功能子区覆盖需要被布置和命名的空间；不要求覆盖整个 `rawBoundary`。

WHY：`rawBoundary` 支持多边形。强行矩形化会把凹角、转折和边缘空间错误切掉，导致后续布置失去墙面资源或误占通道。

**1d. 可行性验证**：验证每个功能子区是否能承载其核心家具或功能，并检查有意留白是否仍保持通畅。

如果某方案只有通过压缩主通道、消灭入口缓冲、遮断采光轴或制造无意窄缝才能成立，则该方案不可行。

### 步骤 2：定义功能（语义层面）

**2a. 功能分配审视**：审视各功能子区的墙面资源和空间品质（采光、动线位置、与相邻空间的关系），决定功能身份；同时说明未生成子区的有意留白承担什么作用。

即使结论是不创建 subZones，也必须输出语义功能带。语义功能带说明空间内部哪些连续区域承担核心功能、过渡功能、可选功能或有意留白；如果父 zone 带有 `optionalTags`，必须审视这些可选功能是否应归属到某个功能带，还是应在本轮省略。

空间阅读三要素：
- **动线方向**：入口 → 各功能点，人的自然行进路径
- **纵深层次**：前场（靠入口，适合过渡/公共功能）vs 远端（适合私密/静态功能）
- **采光轴**：窗侧适合日间活动、需要自然光的功能；高大家具不应切断光路

分区标签一旦确定，后续房间策略只在标签范围内思考——分区决策锁定了后续所有可能性。有意留白不映射 tags，也不作为后续家具布置目标。

**子空间/功能带内的功能方案分歧**：当某个子空间或语义功能带允许两种以上功能方案（如"核心功能-only" vs "核心功能+可选功能"），且方案间对日常使用有质的差异时——这是战略选择，需要在步骤 2b 中一并向用户展示。"区域内的标签分配"不是跳过询问的理由：标签决定了后续房间策略能放什么家具，影响与分区划分同等重大。

WHY：局部最优 ≠ 全局最优。某块延伸区看起来适合做 X，不代表 X 放在这里就是全局最优解。

**2b. 展示分区方案 / 自动代决**：

- 当前执行者具备 `AskUserQuestion`：向用户展示分区分析和推荐方案
- 当前执行者不具备 `AskUserQuestion`：按推荐方案继续，并在最终结果中上报“分区自动代决”

展示内容：
- 推荐方案：功能子区边界或语义功能带 + 各子空间/功能带定位 + 可选功能归属 + 有意留白位置与目的 + 对用户日常使用的具体影响
- 替代方案：如存在多个可行方案，说明核心差异和取舍
- 推荐理由

WHY：分区是整个布局的骨架——边界、功能定位和关键留白一旦确定，锁定后续所有可能性。主控 Agent 应在这里与用户确认；layout-agent 则按推荐方案自动推进，优先保证后台任务能完整跑通。

**2c. 确定功能**：选择推荐方案，功能子区映射到 Zone Tags，Tags 映射到房间策略文件；有意留白只写入分区意图，不生成 tags。
WHY：功能驱动产出"有使用场景的空间"；几何驱动产出"零散家具的堆放区"。

### 步骤 3：输出 subZones（数据层面）
- 修改 `schemes/zones.json` 中父 zone 的 `subZones` 字段
- 每个功能子 zone：`type = "designable"`，`id = "dz_{n}"`
- 子 zone 的 `roomId` 继承父 zone，`tags ⊆ 父 zone tags + optionalTags`
- 子 zone 的 `rawBoundary` 可以是正交多边形；不要为了凑 4 点矩形而牺牲建筑贴合、主通道或关键留白
- 有意留白不强制生成子 zone；若生成了子 zone，就必须有可布置、可命名、可映射 tags 的功能身份
- 写入后 Server 自动创建子目录
- **【必须】**一旦父 zone 产生 `subZones`，父 zone 只作为容器；后续模块读写以叶子 zone 为目标
- 写入后调用 `get_zone_boundaries(zoneIds=[新子zone列表])` — 返回子 zone 边界中的 passage 段（通向相邻分区或开放留白的连通），后续房间 Skill 据此避免在通道处放大型家具

> WHY：子分区的 rawBoundary 不区分墙和通道。passage 段解决了"这条边到底是实墙还是开放连通"的关键问题。

## 三、关键约束

**【硬约束】**
- ① 子空间边界不重叠 — 一个位置不能同时属于两个功能区
- ② 功能子区必须覆盖需要被布置和命名的空间；通道、入口缓冲、采光留白、操作净空可作为有意留白，不强制生成 designable 子区
- ③ 功能定义必须具体（"更衣区"✓ "过渡区"✗）— 模糊功能无法映射 Tags
- ④ 子分区完成后必须调用 get_zone_boundaries — passage 数据是后续房间 Skill 正确工作的前提
- ⑤ 不创建 subZones 仍必须输出语义功能带；"不分区"不能省略功能方案分歧判断

**【软指导】**
- 分区线需有预布置根据 — 不能说明主要家具如何容纳的分区线，只是任意切割
- 异形区优先搜索可选家具位置 — 独特尺寸恰好适配特定家具
- 动线区域保持通透 — 通行功能优先于放置功能
- 子空间边界优先贴合建筑轮廓和功能占用 — 在不增加无意义复杂度的前提下，可使用正交多边形贴合 L/T/凹角空间

**【自由区域】**
- 分区线的精确位置
- 功能区面积比例分配
- 子空间命名风格
- 分区数量（2个还是3个，根据空间实际情况判断）
